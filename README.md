### 4. 使用 Issues 和 Projects 跟踪任务

GitHub 的 Issues 和 Projects 功能可以帮助团队跟踪任务进度、分配任务以及讨论具体问题。

- **Issues**：用于报告 bug、提出新特性或者任何其他需要解决的问题。
- **Projects**：可视化的看板工具，用来组织和优先级排序任务。

鼓励组员创建 Issues 来记录待办事项，并通过评论交流解决方案。

### 5. 开始协作

一旦所有准备工作完成，组员们就可以根据提供的开发指南开始协作了。他们可以通过以下步骤参与开发：

- 克隆仓库并切换到 `dev` 分支。
- 创建自己的功能分支进行开发。
- 完成开发后，提交更改并向 `dev` 分支发起 Pull Request。
- 参与代码审查过程，确保代码质量。

遵循这些步骤，你们就可以高效地进行协同开发了。如果有任何疑问或需要进一步的帮助，请随时提问！
